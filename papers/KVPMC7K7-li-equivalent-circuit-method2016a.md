## § 1 — 研究问题与重要性

**论文直接陈述的研究问题**是：怎样在 hardware-in-the-loop（HIL，硬件在环）测试台上，既保留 modular multilevel converter（MMC，模块化多电平换流器）每个 submodule（SM，子模块）的开关状态和电容动态，又在固定的、通常只有数十微秒的系统步长内完成电路求解和大量 I/O 交换。全细节 MMC 含有数量巨大的开关；开关状态频繁改变会触发大规模矩阵重构，常规 electromagnetic transient（EMT，电磁暂态）工具难以满足硬实时截止时间。另一方面，平均模型虽然快，却把一阀内的电容合并并假定 SM 电压均衡，不能单独处理每个 SM，因此不适合验证电容均压、门极脉冲、闭锁和异常门极等控制功能（源 PDF 物理页 1，Abstract 与 Section I）[pdf:E01][pdf:E02]。

这个问题重要，不只是因为“仿真更快”，而是因为 HIL 的结果直接用于制造商控制与保护系统的 factory acceptance test（FAT，工厂验收测试）。论文所处的工程背景是多端 MMC-HVDC 工程：不同站点由不同制造商供货，必须在投运前验证站间控制协同和保护配合。作者称该方法形成的测试台曾接入制造商控制器，覆盖测试清单中的场景，并在两个多端 MMC-HVDC 项目投运前被采用（源 PDF 物理页 2，Section I）[pdf:E03]。

因此，这项工作的价值可以概括为一个三重约束：**模型细节足以让真实控制器“看见”各个 SM，数值精度足以复现系统与故障响应，计算和 I/O 延迟又必须严格落在每个实时步长内**。论文的核心贡献不是追求器件级开关瞬态，而是在控制器 HIL 所需的时间尺度上，把详细 SM 行为压缩成可并行的固定拓扑等效模型（源 PDF 物理页 1–2，Abstract 与 Section I）[pdf:E01][pdf:E03]。

## § 2 — 前人工作与不足

以下是**论文对已有工作的归纳**，不是本卡对外部文献的独立复核。

第一类是 MMC average model（平均模型）[7], [8]。它们把一阀中的所有电容替换为一个等效电容，在 SM 电压平衡的前提下能很好描述系统级行为，也容易实时运行；不足是无法逐个处理 SM，因而不能支撑需要单模块门极、闭锁、均压和故障细节的 HIL（源 PDF 物理页 1，Section I）[pdf:E02]。

第二类沿用 Dommel 离散化思想，把开关表示为随状态变化的电阻，把电容表示为 Norton 形式；相关工作 [10], [11] 又用阀级 Norton equivalent（诺顿等效）降低计算量。第三类是 state-space nodal（SSN，状态空间节点）方法 [12], [13]，它把系统解耦成较小的状态空间子系统并允许各阀并行求解。第四类是 surrogate network（代理网络）[14]，按闭锁、解锁和旁路状态对 SM 分组，为各组构造 Thevenin equivalent（戴维南等效），并映射到 FPGA（源 PDF 物理页 1，Section I）[pdf:E02]。

作者认为这些方法的共同瓶颈仍是**状态依赖的等效网络需要在开关状态变化时重新计算**。即使矩阵已经缩小或阀已经解耦，海量 SM 的状态变化仍把计算资源消耗在“重新形成网络”上。本文改变的假设是：不再把每次门极变化视为大系统拓扑变化，而是固定 SM 的外部等效拓扑，只更新少量受控源和电容状态；这正是其相对 Norton、Thevenin 和代理网络的主要差异（源 PDF 物理页 1 末至物理页 2，Section I）[pdf:E02][pdf:E03]。

需要保留的批评是：论文没有在同一硬件、同一模型、同一误差指标下直接量化上述 prior methods 的速度与精度。因此，“计算负担更小”在本文中主要由算法结构、FPGA 资源占用和自身运行时间支撑，而不是一个完整的横向 benchmark。

## § 3 — 重建作者的思考路径

下面是**基于证据的逆向重建**，不是论文逐句给出的研发日志。

1. HIL 的真正约束是每一步都必须按时完成，而不是离线总运行时间较短。只要某个开关事件导致矩阵重构超时，控制器闭环就失去实时性；所以优化目标应从“加速原始全开关网络”转向“消除状态变化引起的全局重构”（源 PDF 物理页 1，HIL 与 EMT 瓶颈）[pdf:E02]。
2. 控制器又确实需要单个 SM 的状态，不能退回阀级平均模型。于是可被牺牲的是 SM 内部显式开关拓扑，而不能牺牲的是端口电压、阀电流对电容充放电的作用以及每个 SM 的门极状态（源 PDF 物理页 1–2）[pdf:E01][pdf:E04]。
3. 半桥 SM 虽有 insert、bypass、diode、fault 四种模式，但从两个外部端口看，都能写成同一组理想二极管加受控电压源；模式只决定源值和电容是否承受阀电流。这样，开关事件就从“改变网络连接”变成“改变查表结果”（源 PDF 物理页 2，Figs. 1–5；物理页 3，Tables I–II）[pdf:E04][pdf:E05]。
4. 一阀是大量相同 SM 串联。各 SM 的电容更新只依赖历史电容电压、当前阀电流和自身门极状态，因而可以独立并行；最后只需把同方向的源电压求和，向主系统暴露一个固定的阀级端口（源 PDF 物理页 3，Eq. (2)、Fig. 6 与 Eq. (3)）[pdf:E06][pdf:E07]。
5. FPGA 恰好适合复制大量相同计算核，并能以远小于 CPU 系统步长的周期处理门极事件。于是自然形成多速率分工：CPU 解电网与换流器外部网络，FPGA 解每个 SM，并通过“整步平均阀电压”而不是同步瞬时值耦合两个时间尺度（源 PDF 物理页 4–5，Figs. 8–9 与 Eqs. (4), (6), (7)）[pdf:E08][pdf:E09][pdf:E10]。

这条路径的关键洞察是：**把并行性建立在“SM 状态更新彼此独立”上，而不是仅把一个仍会变拓扑的大矩阵切成几块**。

## § 4 — 核心 Intuition

核心不是更快地求解原来的全开关电路，而是让门极变化不再改变主系统拓扑：每个 SM 始终表现为同一种二极管—受控电压源端口，状态变化只改源值和充电系数（源 PDF 物理页 2，Fig. 5；物理页 3，Tables I–II）[pdf:E04][pdf:E05]。各 SM 用已有状态独立更新，再把源电压相加形成阀级等效，因此可以按 SM 数量横向复制 CPU core 或 FPGA 计算单元（源 PDF 物理页 3，Eq. (3)）[pdf:E07]。FPGA 在细步长吸收快速门极事件，CPU 只接收粗步长内的平均等效电压，从而把高频开关细节隔离在主网络求解之外（源 PDF 物理页 4–5，Fig. 9 与 Eqs. (6)–(7)）[pdf:E09][pdf:E10]。

## § 5 — 具体方法与完整 Pipeline

以论文的 half-bridge MMC valve 为例，一次完整时间推进如下。这里的“真实例子”不是抽象软件 pipeline，而是论文 Figs. 1–9 所示、可直接连接外部控制器的 SM—阀—CPU/FPGA 计算链（源 PDF 物理页 2–5）[pdf:E04][pdf:E08]。

1. **输入与状态。** 每个 SM 保存上一时刻电容电压，接收上、下 IGBT 门极状态以及阀电流方向；公共参数包括 SM 电容、放电电阻和器件正向压降。半桥 SM 被归入 insert、bypass、diode 或 fault 模式；fault 模式把电容快速放电至零，其余模式由阀电流是否流经电容决定充电系数（源 PDF 物理页 2，Figs. 1–5）[pdf:E04]。
2. **模式查表。** 根据门极状态和电流方向，从 Table I 取得正、负方向受控源电压，从 Table II 取得半桥的系数：insert 时电容参与电流，bypass 时不参与，diode 模式是否参与取决于电流方向（源 PDF 物理页 3，Tables I–II）[pdf:E05]。这一步代替了开关动作后的拓扑重构。
3. **更新单个 SM。** CPU 版本用 forward Euler（前向欧拉）在系统步长上更新电容；FPGA 版本则在每个 250 ns 子步上更新门极状态和电容。每个 SM 只读公共阀电流和自己的历史状态，没有 SM 间递归依赖，所以大量实例可完全并行（源 PDF 物理页 3，Eq. (2)；物理页 4，Eq. (4)）[pdf:E06][pdf:E09]。
4. **聚合成阀级端口。** 对每个电流方向，把所有 SM 的受控源电压求和，得到阀级的两个候选源；理想二极管根据阀电流方向选择实际端口电压。主系统因此只看见固定拓扑的两二极管、两受控源等效阀（源 PDF 物理页 3，Fig. 6 与 Eq. (3)）[pdf:E07]。
5. **CPU–FPGA 多速率交换。** CPU 通过 PCIe 在每个 30 μs 同步点把阀电流发送给 FPGA，并在一个 CPU 步内保持该值；FPGA 处理这一粗步中的全部门极子事件。反向传输时，FPGA 不发送同步瞬间的阀电压，而发送该 CPU 步内的平均阀电压，以避免短于 CPU 步长的 insert/bypass 脉冲被漏采（源 PDF 物理页 4，Figs. 8–9；物理页 5，Eq. (6)）[pdf:E08][pdf:E09][pdf:E10]。
6. **闭环 I/O。** 外部控制器的测量和命令可通过铜缆或光纤接入；I/O protocol driver 与阀模型可放在同一 FPGA，论文报告模型到 I/O 的延迟低于 1 μs。对主网络而言，平均阀电压保持了粗步长内的电压时间积分，进而保持臂电流积分的主要驱动量（源 PDF 物理页 5，Eq. (8) 前后）[pdf:E11]。
7. **映射与扩展。** 论文使用 Simulink 的 Xilinx Blockset 和 Xilinx ISE 生成 bitstream。一个含 12 个 MMC 阀、每阀 256 个 SM 的 Virtex-7 实现占用 14% slice registers、22% LUT 和 14% memory；同一固定端口形式还可用于 full-bridge SM（FBSM，全桥子模块）和 double-clamp SM（DCSM，双箝位子模块），但需改写源电压和系数查表，论文给出了 FBSM 的 Tables III–IV，未给出 DCSM 表（源 PDF 物理页 4，FPGA 实现；物理页 5–6，Fig. 11 与 Tables III–IV）[pdf:E08][pdf:E12][pdf:E13]。

论文**未报告** FPGA 的定点字长、量化标度、舍入/饱和策略、pipeline 深度以及实时 CPU 的具体型号；这些不能从资源比例或参考机的 Intel i5 信息反推。

## § 6 — 核心数学推导（无形式化数学则跳过）

这篇论文的数学核心不是求一个新的全局网络方程，而是把 SM 的状态方程、阀级聚合和多速率接口拆成可并行的显式计算。

**第一步：从电荷守恒得到 SM 电容方程。** 对非 fault 模式，电容电流由阀电流贡献和放电电阻贡献组成：

\[
V_c=\frac{1}{C_{SM}}\int i_c\,dt
=\frac{1}{C_{SM}}\int\left(k\,i_{val}-\frac{V_c}{R_{disc}}\right)dt.
\]

其中，\(V_c\) 是电容电压，\(C_{SM}\) 是 SM 电容，\(i_{val}\) 是阀电流，\(R_{disc}\) 是放电电阻。对半桥，\(k\) 由门极状态和电流方向决定；对全桥，Table IV 允许 \(k\) 取正、零或负值，从而表示正向充电、旁路或反向充电（源 PDF 物理页 3，Eq. (1)、Tables I–II；物理页 6，Table IV）[pdf:E05][pdf:E13]。

**第二步：用显式离散避免迭代和矩阵求解。** 论文采用 forward Euler：

\[
V_c(t+T_s)=V_c(t)+\frac{T_s}{C_{SM}}
\left(k(t)i_{val}(t)-\frac{V_c(t)}{R_{disc}}\right).
\]

因为右端只含当前或历史可用量，各 SM 可以互不等待地并行计算（源 PDF 物理页 3，Eq. (2)）[pdf:E06]。工程上要注意，forward Euler 的稳定性仍取决于步长相对电路时间常数；论文称其简单且稳定，但没有给出步长稳定域、截断误差上界或 fault 切换时的形式化误差证明。

**第三步：把串联 SM 压缩为阀级源。** 同方向的理想二极管可合并，串联受控源可直接相加：

\[
V_{sx\Sigma}=\sum_{i=1}^{N}V_{sx-i},\qquad x\in\{p,n\}.
\]

这里 \(N\) 是每阀 SM 数量，\(p\) 和 \(n\) 对应正、负电流方向。这个求和是 reduction（归约）操作；除最后归约外，所有 \(V_{sx-i}\) 都可独立求得（源 PDF 物理页 3，Fig. 6 与 Eq. (3)）[pdf:E07]。

**第四步：在 FPGA 子步内保留门极事件。** 若一个 CPU 步包含 \(r=T_{CPU}/T_{FPGA}\) 个 FPGA 子步，论文把阀电流在该 CPU 步中保持为 \(i_{val}(t)\)，但允许 \(k\) 和 \(V_c\) 每个子步更新：

\[
V_c(t+T_{CPU})=V_c(t)+\frac{T_{FPGA}}{C_{SM}}
\sum_{j=0}^{r-1}\left[
 k(t+jT_{FPGA})i_{val}(t)
 -\frac{V_c(t+jT_{FPGA})}{R_{disc}}
\right].
\]

这就是 FPGA 比纯 CPU 版本更精细的来源：门极状态在子步内可变化，电容放电项也随子步电压更新；但阀电流仍是粗步长冻结量（源 PDF 物理页 4，Eq. (4) 与相邻正文）[pdf:E09]。

**第五步：用平均阀电压守住粗步长积分。** FPGA 返回的不是最后一个子步的瞬时值，而是

\[
\overline{V}_{sx\Sigma}(t+T_{CPU})
=\frac{1}{r}\sum_{j=1}^{r}V_{sx\Sigma}(t+jT_{FPGA}).
\]

因此有

\[
\int_t^{t+T_{CPU}}V_{sx\Sigma}\,dt
=\overline{V}_{sx\Sigma}(t+T_{CPU})T_{CPU}
=\sum_{j=1}^{r}V_{sx\Sigma}(t+jT_{FPGA})T_{FPGA}.
\]

也就是说，CPU 用一个常值代表整步时，其矩形积分与 FPGA 子步积分一致（源 PDF 物理页 5，Eqs. (6)–(7)）[pdf:E10]。这对臂电流尤其重要，因为论文写出的关系为

\[
i_{val}=\frac{1}{L}\int\left(v_{dc+}-v_a-v_{mmc}\right)dt,
\]

其中 \(v_{mmc}\) 正是等效阀输出；保持它的时间积分，比只传同步点瞬时值更直接地保持臂电流变化（源 PDF 物理页 5，Eq. (8)）[pdf:E11]。

数学上最值得注意的边界是：Eq. (7) 证明的是**给定 FPGA 阀电压序列后，粗细两种表示的电压积分相同**，并没有证明该序列在快速变化阀电流下仍接近真实系统。后一个问题取决于 Eq. (4) 的跨步电流冻结假设，也是本文最薄弱之处。

## § 7 — 实验设计与结论

**验证基线与工况。** 测试系统是模仿 France–Spain HVDC link 的双端、单极、101-level half-bridge MMC。每阀 100 个 SM，额定功率 1000 MVA，直流电压 ±320 kV，直流线路 70 km；控制器包含 PQ control、circulating current suppression control（CCSC，环流抑制）和 voltage balancing control（VBC，电压均衡）。CPU test model 用 RTLAB Artemis、步长 30 μs；FPGA test model 的阀为 250 ns、其余系统为 30 μs。参考模型是 EMTP-RV 的 detailed equivalent model，以 30 μs 运行，但可在不连续点回退半步并迭代；该参考模型此前与 full-detail model 比较过（源 PDF 物理页 6，Table V 与 Sections V-A–V-C）[pdf:E14]。

- **问题：模型能否处理二极管自然换流这一最不利的离散事件？ → 实验：** 所有 SM 电容初值为零，脉冲闭锁，交流侧通过 1 kΩ 充电电阻和二极管充电；在 0.22746 s 附近观察上臂阀电压越过总电容电压后的二极管导通。**→ 答案：** EMTP-RV 在事件处把 30 μs 临时减为 15 μs并迭代，而 CPU/FPGA 实时模型维持 30 μs；三者总体重合，实时模型的阀电压过冲为 0.005 pu，持续一个时间步（源 PDF 物理页 6–7，Figs. 12–13）[pdf:E14][pdf:E15]。
- **问题：控制异常和局部闭锁时是否仍一致？ → 实验：** 先将 station 1 的 CCSC 停用 100 ms，再将 phase-a 上臂一个阀闭锁 100 ms，同时其余五阀正常运行且不加专门保护。**→ 答案：** 作者报告 CPU、FPGA 与参考模型的阀电流、平均电容电压、环流和交流电压高度一致（源 PDF 物理页 7，Figs. 14–15）[pdf:E16]。
- **问题：外部交流故障能否正确传播到阀和电容动态？ → 实验：** station 2 并网点施加三相接地故障，200 ms 后清除。**→ 答案：** 三种模型的阀电压、阀电流和平均电容电压波形呈良好一致（源 PDF 物理页 7–8，Fig. 16 与相邻正文）[pdf:E16][pdf:E17]。
- **问题：严重直流故障下的峰值和振荡是否保持？ → 实验：** 在 station 2 邻近处施加永久 pole-to-pole DC fault（极间直流故障），400 μs 后闭锁全部 IGBT，两个交流断路器在两个周波后打开。**→ 答案：** 三种模型的直流电缆电压和电流振荡模式一致；CPU 模型与参考模型的直流故障电流峰值分别为 6.12 pu 和 6.14 pu，差异为 0.33%（源 PDF 物理页 8，Fig. 17 与 Section V-C-5）[pdf:E18]。
- **问题：详细度能否在硬实时截止时间内实现？ → 实验：** CPU 和 FPGA test model 都分布在四个 CPU core；比较每个 core 的执行时间。**→ 答案：** CPU test model 中，每站 600 个 SM 由一个 core 计算需约 35 μs，超过 30 μs 步长；作者称改成每 core 300 个 SM、六 core 后可实时。FPGA test model 中两个 station core 各需 9 μs，controller core 各 18 μs，均低于 30 μs（源 PDF 物理页 8，Table VI）[pdf:E19]。
- **问题：规模扩大后性能如何？ → 实验与报告：** 作者给出 Virtex-7 VC707 的容量估计和离线参考模型运行时间。**→ 答案：** 一块板可模拟超过 6000 个 SM，因而覆盖每阀 1000 个 SM 的大系统；若把低层控制也放入 FPGA，仅处理高层控制的 CPU core 可低至 8 μs。作为参照，Intel i5 2.6 GHz 上的 EMTP-RV 用 142 s 模拟 3.5 s，即约慢于实时 40 倍（源 PDF 物理页 8，Section VI）[pdf:E20]。

**实验支持了什么。** 在给定的 101-level、100 SM/valve、30 μs 主步长和所选控制/故障场景内，等效方法能够让 CPU/FPGA 波形接近 EMTP-RV 参考，并且 FPGA 版本有明确的实时余量。结论部分进一步声称该方法可用于很大的系统，例如五端、每端 \(250\times6\) 个 SM，并已用于实际项目的控制器 HIL（源 PDF 物理页 8–9，Conclusion）[pdf:E21][pdf:E22]。

**不能外推的范围。** 量化验证仍主要是 simulation-to-simulation：参考模型是 EMTP-RV detailed equivalent model，而不是测量波形；图中多数结论依赖视觉重合，系统性误差指标只有少数事件数字。论文没有扫描臂电感、CPU/FPGA 步长比、开关事件相位、定点字长或网络强弱，也不覆盖器件开关损耗、EMI 和半导体瞬态。因此，证据足以支持“该参数点和这些场景下可实时且相近”，不足以证明任意 MMC 尺度和任意快速故障下都保持同一精度。

## § 8 — Take-aways

**5 句话：** ① 本文把 MMC 实时仿真的主要瓶颈从“海量开关导致的大矩阵重构”改写为“固定端口模型上的本地状态更新”（源 PDF 物理页 1–3）[pdf:E02][pdf:E03][pdf:E07]。② 每个 SM 的电容和受控源可独立计算，阀级只做求和，因此算法结构天然适配多核 CPU 和 FPGA。③ 多速率 FPGA 版本以 250 ns 处理门极事件、以 30 μs 同 CPU 交换，并用整步平均阀电压保持粗细步长的电压积分（源 PDF 物理页 4–6）[pdf:E09][pdf:E10][pdf:E14]。④ 在所测二极管换流、控制停用、单阀闭锁、交流故障和直流故障中，结果与 EMTP-RV 参考接近，其中极间故障峰值差异为 0.33%（源 PDF 物理页 7–8）[pdf:E15][pdf:E16][pdf:E17][pdf:E18]。⑤ FPGA 版本在论文的四核分布中有实时余量，但其精度依赖粗步长内阀电流变化足够平缓，这一点没有被系统扫描（源 PDF 物理页 4、8）[pdf:E08][pdf:E20]。

**3 句话：** ① 方法的关键是固定 SM 等效拓扑，把门极事件转成查表和显式积分。② FPGA 负责细粒度 SM 动态，CPU 负责主网络，平均电压接口解决了粗细步长的积分一致性。③ 论文证明了一个有工程价值的可行实现，但没有给出跨参数的误差保证。

**1 句话：** 这篇论文最重要的贡献，是用“固定端口等效 + SM 级并行 + 多速率平均接口”把详细 MMC 控制器 HIL 从离线 EMT 问题变成可按实时截止时间调度的计算问题（源 PDF 物理页 3–5，Eqs. (2)–(7)）[pdf:E06][pdf:E07][pdf:E09][pdf:E10]。

## § 9 — 最脆弱的假设

**最脆弱的假设是：在一个 CPU 步长内，阀电流可以冻结，而不会显著破坏每个 SM 的电荷更新。** 论文明确说明 CPU 在同步点把阀电流发送给 FPGA，并在下一同步点前保持常值；其理由是臂电感使阀电流逐渐变化。与此同时，门极状态和电容电压仍以 FPGA 子步更新（源 PDF 物理页 4，Fig. 8 与 Eq. (4) 前后）[pdf:E08][pdf:E09]。

这个假设之所以比“有足够 FPGA 资源”更关键，是因为并行资源只能加快计算，不能修复错误输入。若故障、弱臂电感、强电网或事件相位使阀电流在 30 μs 内显著弯折，Eq. (4) 会在全部 FPGA 子步中反复使用过时的 \(i_{val}(t)\)。Eq. (6)–(7) 虽能保证**已经算出的阀电压序列**在 CPU 步内积分一致，却不能保证该序列对应正确的电容充放电。基于 Eq. (4) 的直接推断是：电流变化越快、CPU 步越长，冻结电流带来的单步电荷误差越大；快速门极采样本身无法消除这一误差（源 PDF 物理页 4–5）[pdf:E09][pdf:E10]。

论文给出的支持是有限但有价值的：验证系统采用 48.9 mH 臂电感和 30 μs 主步长，在二极管不连续点只观察到 0.005 pu、一个步长的过冲，并在多类故障中保持波形接近（源 PDF 物理页 6–8）[pdf:E14][pdf:E15][pdf:E18]。缺失的证据是针对臂电感、网络强度、故障相位和步长比的敏感性扫描，以及一个把电流冻结误差与允许 HIL 误差直接连接起来的上界。若这些压力测试下误差失控，论文最核心的“细步 FPGA + 粗步 CPU 仍保持准确”会首先失效。

## § 10 — 最小复现实验

一周内不必复现双端 HVDC 和完整控制器，最小实验应只验证本文的核心机制：**固定拓扑 SM 更新、FPGA 子步门极处理和平均电压接口，是否比纯 CPU 粗步模型更接近细步参考。**

- **数据与电路：** 搭建一个 phase-a 上臂，采用论文的 100 个 half-bridge SM、3.3 mF 电容和 48.9 mH 臂电感；使用论文的 30 μs CPU 步长与 250 ns FPGA 子步（源 PDF 物理页 6，Table V 与 test model 设置）[pdf:E14]。输入不是外部 dataset，而是一组可重复的门极序列和二极管充电事件；让事件在 CPU 步内不同相位发生。
- **实现三条路径：** 细步单速参考模型；只在 30 μs 更新的 CPU 等效模型；按 Eq. (4) 子步更新并按 Eqs. (6)–(7)回传平均电压的 FPGA-like 多速率模型（源 PDF 物理页 4–5）[pdf:E09][pdf:E10]。
- **测量：** 每个 CPU 步的阀电压积分、各 SM 电容电压最大误差和均方误差、臂电流误差、事件过冲持续时间，以及实现的每步执行时间。
- **支持判据：** 多速率平均接口在所有事件相位下都比 CPU 粗步模型更接近细步参考；在论文的二极管工况附近复现约 0.005 pu、最多一个 30 μs 步长的过冲量级；同时 100-SM 阀更新和聚合不超出实时截止时间（源 PDF 物理页 7，Fig. 13 与正文）[pdf:E15]。
- **反驳判据：** 平均接口不能稳定降低电容与臂电流误差，或者误差对事件在粗步内的相位高度敏感，或者计算虽并行但仍频繁错过截止时间。任何一项都说明论文的核心 claim 只在更窄条件下成立。

这个实验的优点是把参考精度、接口机制和实时性分开测量，不需要先复现 PQ、CCSC、VBC、双端线路和全部故障逻辑。

## § 11 — 最强反例设计

最有力的反例不是再找一个普通故障，而是**专门破坏“CPU 步内阀电流近似常值”这一耦合假设**。

构造方法如下：保留相同 SM 数量和控制逻辑，逐级减小臂电感或增强外部电压突变，使阀电流在一个 30 μs CPU 步内出现明显斜率和曲率；把二极管换流或直流故障安排在同步点刚过之后，使 FPGA 在几乎整个粗步中都使用事件前的电流样本。用一个全系统细步参考同时更新电流与门极，再与论文式“电流冻结、门极细步、平均电压回传”模型比较。攻击指标不是只看阀电压是否视觉重合，而是看单个 SM 电荷误差是否累积成电容能量漂移、环流偏差或错误的保护触发时刻。

若平均电压仍保持 Eq. (7) 的积分恒等式，但电容电压和臂电流显著偏离参考，就得到一个直接的替代解释：论文中的良好结果可能主要来自所选 48.9 mH 臂电感、30 μs 步长和相对平滑的电流，而不代表多速率接口对任意快速暂态都稳健（源 PDF 物理页 4、6–7）[pdf:E08][pdf:E14][pdf:E15]。这个反例真正挑战的是核心机制，因为它即使给 FPGA 无限并行资源也不会自动消失；要解决它，必须改变 CPU–FPGA 耦合信息，而不只是增加 SM 计算单元。

## § 12 — Follow-up Research Idea

**候选想法：面向硬实时 HIL 的事件感知、能量一致 CPU–FPGA 接口。** 由于本任务只使用源 PDF，未补充检索相关工作，因此这里不声称 novelty。

**（a）未满足需求。** 现有接口每个 CPU 步只向 FPGA 提供一个冻结阀电流，再从 FPGA 取一个平均阀电压。它在论文参数点上有效，但对步内快速电流变化没有误差保证；全局缩小 CPU 步长又可能破坏实时截止时间（源 PDF 物理页 4–5，Figs. 8–9 与 Eqs. (4)–(7)）[pdf:E08][pdf:E09][pdf:E10]。

**（b）为什么可能有本领域认可的价值。** 电力电子 HIL 的高影响结果通常同时要求可实现的实时平台、严格的故障验证和可解释的精度边界。把目标从“典型工况下波形接近”提升为“在固定算力和截止时间下，对最坏事件仍保持能量与电荷误差受控”，会直接提高保护与控制器验收的可信度，而不是只增加可模拟 SM 数量。

**（c）可借鉴的相邻工具。** 候选接口不再只传一个样本，而是在每个同步周期交换阀电流的局部预测信息、门极事件时间戳以及电压/电荷的整步矩；同步点用守恒残差校正下一步状态，并保持全局步长固定。可以把这理解为 numerical co-simulation（数值共仿真）中的守恒耦合思想与 FPGA event timestamping（事件时间戳）的结合；这里仅作为研究工具候选，不把它写成已有文献结论。

**（d）第一个可证伪实验。** 在相同 FPGA 资源和相同 CPU 截止时间下，扫描臂电感、故障相位和 CPU/FPGA 步长比；比较原始冻结接口与候选接口的最坏单步电荷误差、长期电容能量漂移、保护动作时刻偏差和 deadline miss。若候选接口不能在不增加超时的前提下降低最坏误差，想法即被证伪。

**（e）与本文的实质区别。** 本文优化的是 SM 内部表示和并行执行，接口层仍假定粗步内阀电流近似常值；候选工作把**跨速率耦合的正确性**本身作为研究对象，让 FPGA 返回的不只是平均电压，让 CPU 发送的也不只是单点电流。它不是“再加一个预测模块”或“换一块更大 FPGA”，而是把研究问题改写为：怎样在不可缩短的实时步长下，构造对快速不连续事件仍有可验证误差边界的详细 MMC HIL。
